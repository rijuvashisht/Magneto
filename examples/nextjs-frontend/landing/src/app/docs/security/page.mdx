## Risk Levels

| Level | Description | Auto-Execute | Approval |
|-------|-------------|--------------|----------|
| **Low** | Documentation, tests, styling | ✓ | Not required |
| **Medium** | Feature work, refactoring | — | Single approval |
| **High** | Auth, security, deployment | — | Multi approval |

## Security Policies

### Protected Paths

Prevent AI from accessing sensitive files:

```json
{
  "security": {
    "protectedPaths": [
      ".env",
      "src/config/secrets.ts",
      "credentials.json",
      "**/*.key"
    ]
  }
}
```

### Blocked Actions

Prohibit dangerous operations:

```json
{
  "security": {
    "blockedActions": [
      "rm -rf /",
      "drop database",
      "delete production"
    ]
  }
}
```

### Approval Gates

Require explicit approval for critical operations:

```json
{
  "security": {
    "approvalRequired": [
      "production-deploy",
      "database-migration",
      "auth-changes"
    ]
  }
}
```

## Execution Modes

### Observe Mode

Read-only analysis, no modifications:

```bash
magneto run task.md --mode observe
```

Safe for:
- Code review
- Architecture analysis
- Documentation generation

### Assist Mode (Default)

Suggests changes, requires approval:

```bash
magneto run task.md --mode assist
```

Recommended for:
- Daily development
- New feature work
- Code refactoring

### Execute Mode

Auto-executes low-risk tasks:

```bash
magneto run task.md --mode execute
```

Use with caution:
- Tests only in CI
- Automated formatting
- Documentation updates

### Restricted Mode

Maximum oversight:

```bash
magneto run task.md --mode restricted
```

Required for:
- Production systems
- Security-critical code
- Compliance environments

## Interactive Approval

Review each AI action before execution:

```bash
magneto run task.md --interactive
```

Approval workflow:
1. AI proposes change
2. Diff displayed
3. You approve/reject/modify
4. Only then executed

## Security Checklist
