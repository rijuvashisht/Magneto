# Getting Started

Welcome to **Magneto AI** — the multi-agent AI control plane that ships features 3–5× faster, catches security issues before they reach production, and keeps your source code under your control.

## Quick Start (60 seconds)

```bash
# 1. Install globally
npm install -g magneto-ai

# 2. Initialize — auto-detects your stack, or pass packs explicitly
cd your-project
magneto init                                    # auto-detect everything
magneto init --with typescript nextjs           # Next.js project
magneto init --with python fastapi              # Python / FastAPI
magneto init --with java spring-boot            # Java / Spring Boot
magneto init --with aws                         # Terraform / AWS infra

# 3. Plan a task (write a .md task file first, or use an example)
magneto plan tasks/my-task.md

# 4. Run with streaming output
magneto run tasks/my-task.md --stream
```

---

## What Magneto Actually Does

Raw AI assistants load whatever files look relevant and ask you to clarify. Magneto **classifies** your task, **scopes** exactly the files that matter, runs **3–4 specialised agents in parallel**, checks **security guardrails before touching anything**, and returns a merged, deduplicated report.

| Without Magneto | With Magneto |
|---|---|
| 15–40 files loaded (shotgun) | 4–8 files (task-classified) |
| 5–10 back-and-forth exchanges | 0 — all context pre-scoped |
| Re-explains project every session | Persistent `.magneto/memory/` |
| 1 generic agent pass | 3–4 parallel role-focused agents |
| No security check | Every task evaluated before execution |

### Explicit stack

```bash
magneto init --with typescript nextjs         # Next.js app
magneto init --with python fastapi            # FastAPI service
```

## Step 2 — Write a Task File

Tasks are plain Markdown (`.md`), YAML (`.yaml`), or JSON (`.json`). The simplest form:

```markdown
# Add analytics dashboard

## Description
Add a `/dashboard` page showing daily active users, revenue, and error rate.
Uses the existing `/api/metrics` endpoint.

## Acceptance Criteria
- [ ] Page renders server-side
- [ ] Charts use Recharts
- [ ] All numbers have loading skeletons
- [ ] Unit tests for the data-fetch hook
```

Save it as `tasks/add-dashboard.md`.

---

## Step 3 — Plan

```bash
magneto plan tasks/add-dashboard.md
```

Magneto classifies the task, evaluates security risk, and decomposes into parallel subtasks:

```
[magneto] Classification: feature-implementation
[magneto] Security Risk: low ✓
[magneto] Agents: orchestrator, backend, frontend, tester
[magneto] Subtasks:
[magneto]   1. Design /api/metrics response shape    → backend
[magneto]   2. Build dashboard page + components     → frontend
[magneto]   3. Write unit + integration tests        → tester
[magneto] ✓ Plan saved to .magneto/cache/plan-add-dashboard.json
```

### Execution modes

Control how much authority Magneto has:

| Mode | What it does |
|---|---|
| `--mode observe` | Read-only analysis — no files touched |
| `--mode assist` | Suggestions only — you apply changes *(default)* |
| `--mode execute` | Writes files with guardrails |
| `--mode restricted` | Every action requires explicit approval |

```bash
magneto run tasks/audit.md --mode observe          # safe read-only scan
magneto run tasks/refactor.md --mode execute       # automated with guardrails
magneto run tasks/deploy.md --mode restricted      # approve every step
magneto run tasks/feature.md --approve-each        # interactive per-step
```

## Choosing a Runner

Magneto has six execution backends. It auto-selects based on your environment — or you override with `--runner`.

| Runner | When to use | Data egress | Requires |
|---|---|---|---|
| `openai` | Default cloud runner | API call to OpenAI | `OPENAI_API_KEY` |
| `copilot-local` | GitHub Copilot in your IDE | None | Copilot active |
| `copilot-cloud` | Copilot Cloud endpoint | API call | `MAGNETO_COPILOT_CLOUD_TOKEN` |
| `cascade` | Windsurf / Cascade IDE | None | Windsurf active |
| `gemini` | Google AI models | API call to Google | `GEMINI_API_KEY` |
| `ollama` | Air-gapped / regulated environments | **None** | `ollama serve` running |

### Ollama — zero egress, no API key

For healthcare, finance, or classified environments where source code cannot leave your machine:

```bash
# One-time setup
ollama pull qwen2.5-coder
ollama serve

# Tell Magneto to use Ollama
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=qwen2.5-coder

magneto run tasks/audit.md --runner ollama --stream
```

```
[magneto] Runner:      ollama (qwen2.5-coder @ localhost)
[magneto] Data egress: none ✓ — all processing is local
[magneto] ✓ Task complete  metadata.dataEgress = "none"
```

Every result is tagged with `dataEgress: "none"` — verifiable in `.magneto/audit/approvals.json`.

## Power Packs — Domain Intelligence

Power packs activate stack-specific checks. Each pack ships with real rules, not just suggestions.

### Language packs

| Pack | Checks | What it finds |
|---|---|---|
| `typescript` | 8 | Import cycles, `any` leakage, unsafe casts, missing return types |
| `python` | 14 | Hardcoded secrets, `shell=True`, `eval`, missing timeouts, pickle risk |
| `java` | 14 | Unsafe deserialization, `catch Throwable`, mutable statics, thread safety |

### Framework packs

| Pack | Checks | What it finds |
|---|---|---|
| `nextjs` | 10 | Missing `'use client'`, hydration violations, server/client boundary bugs |
| `fastapi` | 10 | CORS wildcard + credentials, `debug=True`, missing Pydantic validation |
| `spring-boot` | 12 | `open-in-view=true` (N+1), `@Transactional` on private, actuator wildcard |

### Cloud / infra packs

| Pack | Checks | What it finds |
|---|---|---|
| `aws` | 16 | IAM `*`, public S3 ACL, SSH 0.0.0.0/0, unencrypted RDS, hardcoded AKIA keys |
| `azure` | ~12 | RBAC over-permission, public storage, unencrypted disks, NSG wide-open rules |

```bash
# Combine freely
magneto init --with typescript nextjs python fastapi aws
# → 58 checks active across your full stack
```

## Real-World Use Cases

### Pre-deploy security audit

```bash
magneto run tasks/pre-deploy-audit.md --mode observe
# Scans all files, zero writes, flags issues before CI
```

### Feature implementation with agents

```bash
magneto plan tasks/add-payments.md
magneto run tasks/add-payments.md --mode execute --stream
magneto merge .magneto/cache --format markdown
```

### Infrastructure review (AWS)

```bash
magneto init --with aws
magneto run tasks/infra-review.md --mode observe
# Checks 31 .tf files for IAM, S3, SGs, encryption in ~10s
```

### Regulated / air-gapped environment

```bash
magneto run tasks/audit.md --runner ollama
# Full AI reasoning, nothing leaves your network
```

### Auto-fix with approval

```bash
magneto run tasks/refactor.md --mode execute --approve-each
# Presents each proposed change for your sign-off before writing
```
