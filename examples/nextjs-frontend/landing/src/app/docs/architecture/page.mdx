## Layer Details

### 1. CLI Layer

The user interface to Magneto AI. Commands are declarative — you describe what you want, not how to do it.

**Key Commands:**
- `init` — Project scaffolding
- `plan` — Task analysis and classification
- `run` — Multi-agent execution
- `graph` — Knowledge graph operations
- `telepathy` — Auto-discovery

### 2. Orchestration Layer

The control plane that coordinates all AI operations.

**Components:**

| Component | Purpose |
|-----------|---------|
| Task Classifier | Determines task type, complexity, and required roles |
| Agent Spawner | Creates specialized agent instances |
| Result Merger | Consolidates multi-agent outputs |
| Security Engine | Evaluates risks and enforces guardrails |

### 3. Agent Layer

Specialized AI agents with distinct responsibilities:

- **Orchestrator** — Coordinates task decomposition and execution order
- **Backend** — Handles APIs, data models, and server logic
- **Tester** — Generates tests and validates coverage
- **Requirements** — Traces specs to implementation
- **Frontend** — UI components and styling
- **DevOps** — Infrastructure and deployment

### 4. Intelligence Layer

Persistent systems that retain knowledge:

**Knowledge Graph**
- File relationships and dependencies
- Function/class hierarchies
- Import/export mapping
- Community detection for modular architecture

**Memory Store**
- Project decisions and rationale
- Architecture patterns
- Security policies
- Agent learnings

**Checkpoints**
- Execution state persistence
- Resume capability
- Rollback support

**Telepathy Engine**
- Jira/GitHub integration
- Requirements parsing
- Auto-classification
- Continuous discovery

### 5. Runner Layer

AI model interfaces:

- **OpenAI** — GPT-4, GPT-3.5 via API
- **Copilot Local** — GitHub Copilot in IDE
- **Copilot Cloud** — Copilot API endpoint
- **Custom** — Bring your own model

## Extension Points

Magneto AI is designed for extensibility:

**Power Packs**
```bash
.magneto/packs/
├── typescript/
│   ├── rules.md
│   └── pack.json
├── nextjs/
│   ├── rules.md
│   └── pack.json
```

**Adapters**
```typescript
// Custom runner adapter
export class MyAdapter implements RunnerAdapter {
  async execute(input: RunnerInput): Promise<RunnerOutput> {
    // Implementation
  }
}
```

**Hooks**
- Pre-plan
- Pre-execute
- Post-execute
- On-checkpoint
