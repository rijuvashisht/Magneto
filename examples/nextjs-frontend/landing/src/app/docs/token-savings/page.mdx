# Token Savings & ROI

Magneto AI dramatically reduces token usage through intelligent context management, scoped prompts, and memory persistence.

## The Token Problem

Traditional AI-assisted development wastes tokens on:

- **Re-scanning** the same files every session
- **Re-explaining** your architecture repeatedly  
- **Loading irrelevant files** that don't affect the task
- **Re-deriving** context from raw code instead of structured memory

## How Magneto Saves Tokens

### 1. Scoped File Loading

Instead of loading your entire codebase, Magneto loads only relevant files:

```bash
# Traditional AI: "Help me fix a bug"
→ Loads 50 files (150k tokens)

# Magneto: magneto plan tasks/fix-auth-bug.md
→ Loads 4 scoped files (12k tokens)
→ 92% reduction in context tokens
```

### 2. Knowledge Graph Context

Pre-computed relationships mean no re-scanning:

```
Traditional: 
  AI analyzes imports → builds mental model → suggests changes
  (~5k tokens just for context building)

Magneto:
  Knowledge graph provides dependencies instantly
  (~500 tokens for context reference)
```

### 3. Memory Persistence

Decisions and patterns remembered across sessions:

```
Session 1: "We use Zustand for state, not Redux"
→ Stored in memory (one-time 500 tokens)

Session 5: "Add shopping cart"
→ Memory provides: "Use Zustand, store pattern in src/stores/"
→ Zero re-explanation needed
```

### 4. Role-Specific Prompts

Instead of one giant prompt, scoped prompts per role:

```
Traditional: "Implement this feature" (backend + frontend + tests)
→ 1 prompt × 8k tokens = 8k tokens

Magneto: Decomposed into 3 prompts
→ Backend prompt × 3k tokens
→ Frontend prompt × 2k tokens  
→ Test prompt × 2k tokens
→ Total: 7k tokens + better results
```
